# Similarity Engine

Поиск похожих матчей и retrieval-контекст для RAG (Гл. 3, спринт 36).
Контракт — `proto/services.proto` → `SimilarityService`:

- `FindSimilar(entity="match", reference_id, top_k)` — топ-k похожих
  матчей с косинус-скором; `player`/`situation` — следующие фазы
  (UNIMPLEMENTED);
- `RetrieveContext(query_embedding, top_k)` — текстовые сводки ближайших
  матчей («Матч N: победа Dire за 44 мин, сюжет: камбэк…») — прецеденты
  для LLM-Coach.

## Эмбеддинг матча (`engine/embed.py`)

Три L2-нормированных блока: траектория (кривые networth/xp/kills,
ресэмплинг на 12 точек долей длительности, нормировка на суммарную
экономику момента), мета (длительность/убийства/башни), составы
(multi-hot героев обеих сторон; словарь строится из витрины при каждой
перестройке). У JSON-матчей нет PMF-строк → блок составов нулевой,
похожесть считается по траектории.

## Индекс (`engine/index.py`)

In-memory, из витрин ClickHouse, фоновое обновление раз в
REFRESH_INTERVAL_S (300). Точный косинус-поиск: на 10^3–10^4 матчей —
миллисекунды; при 10^5+ интерфейс переживёт замену на FAISS/ANN.
